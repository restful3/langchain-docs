# 철학

LangChain은 LLM으로 구축하기 시작하기 가장 쉬운 장소가 되면서도 동시에 유연하고 프로덕션 준비가 된 상태여야 합니다.

LangChain은 몇 가지 핵심 신념으로 주도됩니다:

*   대형 언어 모델(LLM)은 강력한 새로운 기술입니다.
*   LLM은 외부 데이터 소스와 결합할 때 훨씬 더 우수합니다.
*   LLM은 미래 애플리케이션의 모습을 변형할 것입니다. 특히 미래의 애플리케이션은 점점 더 Agent적일 것입니다.
*   그 변환은 아직 매우 초기 단계입니다.
*   Agent 애플리케이션의 프로토타입을 구축하는 것은 쉽지만, 프로덕션에 배포할 수 있을 만큼 신뢰할 수 있는 Agent를 구축하는 것은 여전히 정말 어렵습니다.

LangChain으로 우리는 두 가지 핵심 초점을 가지고 있습니다:

1.  **우리는 개발자가 최고의 모델로 구축할 수 있도록 지원하려고 합니다.**
    서로 다른 공급자는 서로 다른 API, 서로 다른 모델 매개변수 및 서로 다른 메시지 형식을 노출합니다. 이러한 모델 입력과 출력을 표준화하는 것이 핵심 초점이므로 개발자가 최신 최첨단 모델로 쉽게 변경하고 종속성을 피할 수 있습니다.
2.  **우리는 모델을 사용하여 다른 데이터 및 컴퓨팅과 상호작용하는 더 복잡한 흐름을 쉽게 오케스트레이션할 수 있도록 하려고 합니다.**
    모델은 단지 *텍스트 생성*을 위해서만이 아니라 다른 데이터와 상호작용하는 더 복잡한 흐름을 오케스트레이션하는 데 사용되어야 합니다. LangChain은 LLM이 동적으로 사용할 수 있는 [Tool](/oss/python/langchain/tools)을 정의하고 구조화되지 않은 데이터의 파싱 및 접근을 지원하는 것을 쉽게 만듭니다.

## 역사

현장에서 지속적인 변화 속도를 감안할 때 LangChain도 시간이 지남에 따라 진화했습니다. 다음은 LangChain이 LLM으로 구축하는 것이 무엇을 의미하는지와 함께 진화해온 방식에 대한 간략한 타임라인입니다:

### 2022-10-24
**v0.0.1**

ChatGPT 출시 한 달 전, **LangChain은 Python 패키지로 출시되었습니다**. 두 가지 주요 구성 요소로 구성되어 있습니다:
*   LLM 추상화
*   "Chains", 또는 일반적인 사용 사례를 위해 실행할 미리 정해진 계산 단계. 예: RAG - 검색 단계를 실행한 다음 생성 단계를 실행합니다.

LangChain이라는 이름은 "Language"(언어 모델 같은)와 "Chains"에서 나왔습니다.

### 2022-12
첫 번째 범용 Agent가 LangChain에 추가되었습니다.

이러한 범용 Agent는 [ReAct 논문](https://arxiv.org/abs/2210.03629)(ReAct는 Reasoning and Acting의 약자)을 기반으로 했습니다. 그들은 LLM을 사용하여 Tool 호출을 나타내는 JSON을 생성한 다음 해당 JSON을 파싱하여 호출할 Tool을 결정했습니다.

### 2023-01
OpenAI가 'Chat Completion' API를 출시합니다. 이는 단순한 완료 인터페이스보다는 채팅과 유사한 인터페이스(System, User, Assistant 역할을 포함한 메시지)를 지원하는 첫 번째 API였습니다. LangChain이 이에 대한 지원을 추가했습니다.

### 2023-01
LangChain이 JavaScript 버전을 출시합니다.

### 2023-02
LangChain Inc.가 회사로 설립되었습니다.

### 2023-03
OpenAI가 자신의 API에서 'function calling'을 출시합니다. 이를 통해 LLM은 단순한 JSON보다 더 잘 구조화된 출력을 반환할 수 있었습니다. LangChain이 이에 대한 지원을 추가했습니다.

### 2023-06
LangSmith가 프라이빗 베타로 출시됩니다. LangSmith는 LLM 애플리케이션에 대한 관찰성과 평가를 제공합니다.

### 2024-01
**v0.1.0**

LangChain이 0.1.0을 출시합니다. 동시에 업계는 단순한 프로토타입에서 프로덕션 애플리케이션으로 성숙해지고 있었습니다. 이 릴리스는 LangChain을 더 안정적이고 프로덕션 준비가 된 상태로 만드는 데 중점을 두었습니다.

### 2024-02
LangGraph가 오픈소스 라이브러리로 출시됩니다. LangGraph는 Agent를 구축하기 위한 저수준 오케스트레이션 계층입니다. 일급 스트리밍, 지속 가능한 실행, 단기 메모리 및 인간-루프 상호작용을 지원합니다.

### 2024-06
700개 이상의 통합. LangChain을 더 쉽게 유지 관리하고 사용하기 위해 핵심 패키지에서 통합을 자체 독립 실행형 패키지 또는 `langchain-community`로 분리하기 시작했습니다.

### 2024-10
개발자가 애플리케이션의 신뢰성을 개선하려고 할 때, 높은 수준의 인터페이스가 제공하는 것보다 더 많은 제어가 필요했습니다. LangGraph가 그 저수준 유연성을 제공했습니다. 대부분의 Chain과 Agent는 LangChain에서 사용 중단되었으며, 이들을 LangGraph로 마이그레이션하는 방법에 대한 가이드가 있습니다. LangGraph에서 생성된 높은 수준의 추상화가 여전히 하나 있습니다: Agent 추상화. LangChain의 ReAct Agent와 동일한 인터페이스를 가진 저수준 LangGraph 위에 구축됩니다.

### 2025-04
모델 API가 더 다중 모달이 됩니다.

모델이 파일, 이미지, 비디오 등을 수용하기 시작했습니다. 개발자가 표준 방식으로 이러한 다중 모달 입력을 지정할 수 있도록 `langchain-core` 메시지 형식을 그에 따라 업데이트했습니다.

### 2025-10-20
**v1.0.0**

LangChain이 두 가지 주요 변경 사항으로 1.0을 출시합니다:

1.  **`langchain`의 모든 Chain과 Agent의 완전한 개편.** 모든 Chain과 Agent는 이제 하나의 높은 수준의 추상화로만 대체됩니다: LangGraph 위에 구축된 Agent 추상화. 이는 원래 LangGraph에서 생성되었지만 LangChain으로 이동된 높은 수준의 추상화였습니다. 여전히 업그레이드하지 않으려는 이전 LangChain Chain/Agent를 사용하는 사용자의 경우(참고: 업그레이드할 것을 권장합니다) `langchain-classic` 패키지를 설치하여 이전 LangChain을 계속 사용할 수 있습니다.
2.  **표준 메시지 콘텐츠 형식.** 모델 API는 단순한 콘텐츠 문자열을 가진 메시지를 반환하는 것에서 더 복잡한 출력 유형(추론 블록, 인용, 서버 측 Tool 호출 등)으로 진화했습니다. LangChain은 메시지 형식을 진화시켜 공급자 전체에서 이들을 표준화했습니다.
