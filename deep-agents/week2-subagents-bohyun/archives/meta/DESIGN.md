# 디자인 — 2주차 발표 자료

## 청중 프로파일

| 항목 | 내용 |
|---|---|
| 전제 지식 | LangChain 도구 호출과 `create_deep_agent()` 기본 형태를 알고 있음 |
| 기대 | 서브에이전트를 언제/어떻게 쓰고, 위험한 도구 호출을 어떻게 승인 흐름으로 감싸는지 이해 |
| 시간 | 발표 17분 + Q&A 3분 |
| 톤 | 실습 중심, API 필드는 정확히 짚고 위험 제어는 사례로 설명 |

## 학습 목표

| # | 목표 | 검증 행동 |
|---:|---|---|
| L1 | 서브에이전트의 핵심 가치 설명 | 컨텍스트 격리와 전문화의 차이를 말한다 |
| L2 | SubAgent 딕셔너리 작성 | `name`, `description`, `system_prompt`, `tools`를 채운다 |
| L3 | HITL 기본 구성 | `interrupt_on`과 `checkpointer`가 왜 필요한지 설명한다 |
| L4 | 승인 재개 흐름 구현 | `__interrupt__`를 읽고 `Command(resume=...)`로 재개한다 |
| L5 | 서브에이전트별 승인 정책 설계 | 메인/서브의 `interrupt_on` 차이를 설정한다 |

## 포함 / 제외

포함: SubAgent 구성, 서브에이전트 선택 기준, `interrupt_on`, `MemorySaver`, `thread_id`, `Command(resume=...)`, 서브에이전트 내부 승인 재정의.  
제외: 장기 메모리 심화, 백엔드 샌드박스 심화, 미들웨어 내부 구현 심화.
